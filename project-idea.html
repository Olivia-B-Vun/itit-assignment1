Potential Project Idea

The Problem

Hint: It's Me!

I really like organisation, and everything fits into a neat little box. To facilitate this, I use a range of online and offline tools. Some of the tools I currently use include:

  • Dog Board
  • Google Calendar & Google Sheets
  • House Board
  • My Fitness Pal
  • Stronglifts
  • Trello
  • Whiteboard

As you can imagine, it’s a lot of manual work to keep the information up to date and there are a lot of redundancies. However, the biggest issue is object permanence or rather, the lack thereof. My solution to this is to have one mobile application that does the lot, provides you with relevant notifications but also gives you the option to turn off functionality that isn’t required.

Market Research

Similar apps definitely exist on the market. However, after doing the research, it seems that a majority of these applications only perform on function. Based on this, I had two main take-aways:
The first, many of the foud during my market research were available on both Android and iOS, with only a handful being platform “exclusives” or web applications (Schedule Once & ADHD Tracker 1.0). This meant that someone who utilised multiple devices to organise their daily lives didn’t have to download and install multiple applications to track the same information.
But the main issue that resulted from my market research; which ultimately rendered point one moot; is that an end user who utilised these sorts of tools had to download multiple applications to paint a full picture.

Tools & Skills

Because this application needs to be cross-platform I’m probably not going to want to use something like Swift which is used predominantly in iOS application development or Kotlin for Android.

Instead, I’m going to stick with the devil I (sort of) know and build this app in JavaScript.
Not only is JavaScript relatively robust, React Native is great for cross-platform development. The other upside to this, is that it’s almost as old as I am so, finding a developer with the skills required to create the application will be a piece of cake, and better yet, finding an IDE that can support the language is even easier.
Finally, by using JavaScript, I can then build a Hybrid App and use one codebase for both operating systems resulting in easier maintenance and updates.

MVP

Calendar

This feature would track regular events such as birthday and bills, task due dates (pulled from the o do list). It should have the ability to change the calendar view so that the end user can see hour by hour (or in x minute increments) what my day looks like; thus, allowing me to implement timetabling.

To-Do List

This feature (like a Kanban board on Trello) will allow an end user to track ad-hoc tasks such as organising an appointment and regular tasks such as chores. From there, I would like to be able to set due dates that appear on the calendar and send a push notification to the user's phone

Notifications

Object permanence has been a big factor in this project; so naturally, the ability to have notifications appear on the lock screen of an end user’s phone is a MUST.
